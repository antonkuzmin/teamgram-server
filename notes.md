## Почистить место docker:

docker system prune -a --volumes -f
docker volume prune -a -f

## 2. Полное высвобождение места на Mac
Если prune не помог, значит, виртуальный диск (Docker.raw) уже раздулся. Его нужно заставить сжаться.
Откройте Docker Desktop.
Нажмите на иконку жука (Troubleshoot).
Нажмите Clean / Purge data.
Выберите Disk image и нажмите Delete.
Docker перезапустится и создаст чистый, маленький файл диска.

## 3. Как избежать проблемы в будущем
Используйте -v при удалении: 
Чтобы контейнеры удалялись сразу с томами данных:
docker compose down -v

- Ограничьте размер диска:
Зайдите в Settings -> Resources -> Advanced -> Установите разумный лимит для Disk image size (например, 20-40 GB).

- Регулярная чистка: 
Добавьте в привычку запускать docker system prune -a раз в неделю.

Почему это происходит: 
Вы используете docker compose up --build. Если в Dockerfile есть RUN apt-get update или скачивание больших файлов, каждый билд создает новый слой, который остается в кэше, пока вы его не очистите.
